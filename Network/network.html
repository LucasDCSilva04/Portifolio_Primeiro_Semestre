Redes de Computadores

 As redes de computadores são sistemas de comunicação que permitem a troca de informações e recursos entre dispositivos interconectados, facilitando a comunicação e o compartilhamento de dados em escala local e global.

OSI e TCP/IP

O modelo OSI e o protocolo TCP, são dois conceitos fundamentais em redes de computadores. Eles desempenham papéis distintos, mas complementares, na comunicação e no intercâmbio de dados entre dispositivos em uma rede.
O modelo OSI é amplamente utilizado como referência para entender e projetar redes de comunicação. O TCP é um protocolo de transporte confiável que opera na camada de transporte do modelo OSI. Ele fornece controle de fluxo, controle de congestionamento e garantia de entrega dos pacotes de dados. Ele também lida com a retransmissão de pacotes perdidos ou corrompidos.

➪
Camada Física:

  É a camada mais baixa do modelo OSI e lida com a transmissão física dos dados. Ela define as especificações mecânicas, elétricas e de sinalização dos dispositivos de rede. Essa camada trata do meio físico de transmissão, como cabos, conectores e sinais elétricos.

Camada de Enlace de Dados:

 A camada de enlace de dados fornece a transferência confiável de dados entre nós adjacentes em uma rede. Ela segmenta os dados em quadros, adiciona informações de controle de erros e controle de acesso ao meio, garantindo que a comunicação seja livre de erros e eficiente.

Camada de Rede:

 A camada de rede gerencia o roteamento dos dados em uma rede. Ela trata do encaminhamento dos pacotes de dados entre diferentes redes e determina o melhor caminho para a transmissão. Os protocolos IP (Internet Protocol) são usados nessa camada para endereçamento e roteamento.

Camada de Transporte:

 A camada de transporte é responsável pela transferência confiável de dados entre hosts. Ela divide os dados recebidos da camada superior em segmentos e fornece controle de fluxo e controle de erros para garantir que os dados sejam entregues corretamente e na sequência correta.

Camada de Sessão:

 A camada de sessão estabelece, mantém e encerra sessões de comunicação entre aplicativos em diferentes hosts. Ela permite que as aplicações estabeleçam uma conexão e gerenciem o diálogo entre si.

Camada de Apresentação:

 A camada de apresentação lida com a representação dos dados e seu formato para que possam ser entendidos pelos aplicativos. Ela trata de funções como criptografia, compressão e conversão de formatos de dados.

Camada de Aplicação:

 A camada de aplicação é a camada mais alta do modelo OSI. Ela fornece interfaces e protocolos para que os aplicativos de usuário se comuniquem com a rede. Exemplos de protocolos nessa camada incluem HTTP, SMTP e FTP.

➪
Máscara, Rede e Broadcast

Em resumo, a máscara de rede é usada para definir a porção do endereço IP que representa a rede e o host, enquanto a rede é um conjunto de dispositivos interconectados que trocam informações. O broadcast é uma forma de comunicação em que uma mensagem é enviada para todos os dispositivos em uma rede.

Tipos de Redes

Aqui está uma explicação resumida sobre os tipos de redes, com base em sua escala e alcance geográfico:
PAN: É uma rede de área pessoal que abrange uma área muito pequena, geralmente dentro de um espaço pessoal limitado, como um escritório, sala ou residência. As redes PAN são projetadas para conectar dispositivos pessoais, como smartphones, tablets, laptops e dispositivos domésticos inteligentes. Exemplos de tecnologias usadas em redes PAN incluem Bluetooth e Zigbee.
CAN: É uma rede de área do campus que conecta várias edificações em um local geográfico próximo, como um campus universitário, um parque empresarial ou uma área residencial. As redes CAN são usadas para fornecer conectividade local entre diferentes departamentos, prédios ou instalações dentro de um campus ou área.

LAN: É uma rede de área local que abrange uma área geográfica limitada, como um escritório, uma residência ou um prédio. É utilizada para compartilhar recursos, como impressoras e arquivos, entre os dispositivos conectados.
MAN: É uma rede que abrange uma área metropolitana, como uma cidade ou uma região específica. Pode ser utilizada por organizações governamentais, instituições educacionais ou empresas que precisam interconectar suas filiais e escritórios em uma área geográfica maior que uma LAN.
WAN: É uma rede de área ampla que abrange uma área geográfica extensa, como uma cidade, um país ou até mesmo várias partes do mundo. É usada para conectar LANs distantes e pode envolver tecnologias como linhas telefônicas, cabos submarinos ou redes de satélite.

Meios fisicos de transmissão guiados

Par Trançado: É um tipo de cabo que consiste em dois fios de cobre isolados e entrelaçados entre si. É amplamente utilizado em redes Ethernet e telefonia.
Coaxial: É um cabo que possui um condutor central. O cabo coaxial foi amplamente utilizado em redes de TV a cabo e, em algumas aplicações, ainda é usado para transmissão de dados em redes locais.
Fibra Óptica: Utiliza fibras de vidro ou plástico para transmitir pulsos de luz. Oferece alta velocidade, largura de banda e imunidade a interferências eletromagnéticas. É amplamente utilizado em redes de longa distância.

Meios fisicos de transmissão não guiados

Comunicação por Rádio: Usa ondas de rádio para a transmissão de dados sem fio. É amplamente utilizado em redes Wi-Fi e redes celulares. As ondas de rádio podem cobrir áreas extensas.
Micro-ondas: Utiliza ondas eletromagnéticas de alta frequência para transmitir dados. É comumente usado em comunicações ponto a ponto de longa distância, como em redes de telefonia celular.
Infravermelho: Transmite dados por meio de pulsos de luz infravermelha. É comumente utilizado em dispositivos de controle remoto, mas também pode ser usado para comunicação de curto alcance em ambientes internos.

Transmissão de Dados

Simplex: A comunicação ocorre apenas em uma direção, os dados são transmitidos de um dispositivo para outro, mas não há capacidade de receber dados de volta. Os dados são transmitidos de uma estação para múltiplos receptores, mas não há um canal de retorno para os receptores transmitirem de volta.
Half-duplex: A comunicação pode ocorrer em ambas as direções, mas não simultaneamente. Um dispositivo pode transmitir dados e alternar para o modo de recebimento para receber dados do outro dispositivo. No entanto, os dispositivos não podem transmitir e receber dados ao mesmo tempo. Walkie-talkies são um exemplo comum de comunicação half-duplex.
Full-duplex: A comunicação ocorre simultaneamente em ambas as direções. Isso significa que os dispositivos podem transmitir e receber dados ao mesmo tempo. Em uma comunicação full-duplex, não há necessidade de alternar entre os modos de transmissão e recepção. Um exemplo comum de comunicação full-duplex é uma chamada telefônica, em que ambas as partes podem falar e ouvir simultaneamente.

Topologias

As topologias de rede referem-se à maneira como os dispositivos estão conectados e organizados em uma rede de computadores. Existem várias topologias comuns e aqui está um resumo sobre as principais topologias de rede:
Barramento: Todos os dispositivos são conectados a um único cabo compartilhado, conhecido como barramento. Os dados são transmitidos ao longo do cabo e todos os dispositivos da rede recebem esses dados. No entanto, apenas um dispositivo pode transmitir por vez.
Estrela: Todos os dispositivos são conectados a um dispositivo central, como um switch ou hub, formando uma estrutura em formato de estrela. Cada dispositivo se comunica diretamente com o dispositivo central, que coordena a transmissão de dados entre os dispositivos. Se um dispositivo falhar, os outros dispositivos ainda podem se comunicar normalmente.

Anel: Os dispositivos são conectados formando um circuito fechado em formato de anel. Cada dispositivo está conectado diretamente aos dispositivos vizinhos e os dados são transmitidos em uma única direção ao redor do anel. Cada dispositivo repassa os dados até que o dispositivo de destino seja alcançado.
Malha: Todos os dispositivos estão conectados uns aos outros, formando uma rede totalmente conectada. Cada dispositivo possui uma conexão direta com todos os outros dispositivos da rede. A topologia em malha oferece redundância e alta confiabilidade, pois, se um caminho falhar, ainda existem rotas alternativas para a comunicação.
Árvore: Também conhecida como topologia hierárquica, é uma combinação da topologia em estrela e em barramento. Ela usa uma estrutura em camadas, na qual grupos de dispositivos são conectados a um switch central. Esses switches, por sua vez, são conectados a um switch principal, formando uma hierarquia.

Elementos de uma Rede

Os elementos essenciais de uma rede de computadores são: placa de rede, hub, switch, roteador, repetidor, gateway.
Em resumo, a placa de rede é um componente que permite a conexão de um dispositivo à rede, o hub é um dispositivo simples que replica os dados para todos os dispositivos conectados, o switch encaminha os dados apenas para o dispositivo de destino, o roteador interconecta redes diferentes, o repetidor amplifica o sinal para estender o alcance da rede, e o gateway permite a comunicação entre redes diferentes traduzindo os protocolos. Cada um desses elementos desempenha um papel específico no funcionamento e na conectividade de uma rede de computadores.

Um pouco sobre

Hub: Ele atua como um ponto central de conexão para vários dispositivos em uma rede. Quando um dispositivo envia dados para o hub, ele é transmitido para todos os outros dispositivos conectados ao hub. Isso é conhecido como "broadcasting" e faz com que todos os dispositivos recebam os dados, mesmo que não sejam o destinatário pretendido.
Switch: Um switch é um dispositivo mais avançado que atua como um concentrador de rede. Ele analisa os dados de entrada e os encaminha apenas para o dispositivo de destino apropriado. Isso é feito através do uso de tabelas de endereços MAC, que associam endereços físicos únicos a cada dispositivo conectado.

Roteador: Responsável por encaminhar pacotes de dados entre diferentes redes, como a Internet e uma LAN. Os roteadores usam tabelas de roteamento para determinar o melhor caminho para encaminhar os pacotes de dados com base nas informações contidas nos cabeçalhos dos pacotes. Eles podem executar funções como o NAT, que permite que vários dispositivos compartilhem um único endereço IP público, e também, filtrando pacotes indesejados ou maliciosos.
Nuvem: Entrega de serviços de computação, como armazenamento, processamento de dados e aplicativos, por meio da Internet. Em vez de hospedar esses serviços em servidores locais, eles são fornecidos por provedores de serviços em seus data centers. Os usuários podem acessar e usar esses serviços por meio de uma conexão com a Internet, sem precisar gerenciar fisicamente a infraestrutura subjacente.

DHCP: Protocolo de rede que permite a atribuição automática de endereços IP e outras configurações de rede, como máscara de sub-rede, gateway padrão e servidores DNS. Em vez de configurar manualmente cada dispositivo com essas informações, o DHCP automatiza o processo.
DNS: Traduz nomes de domínio (por exemplo, www.exemplo.com) em endereços IP. Quando você digita um nome de domínio em um navegador da web, o sistema DNS é responsável por localizar o endereço IP associado a esse nome de domínio, permitindo a comunicação entre o seu dispositivo e o servidor web desejado.
